# UX-ревью (июль 2026)

Визуальный осмотр работающей игры (локальный запуск, headless-браузер: два игрока,
наблюдатель, мобильный вьюпорт). Ошибок в консоли за все прогоны — ноль.
Ниже — что стоит улучшить, по убыванию важности.

---

## 1. Туториал показывается поверх живого матча ✅ решено (вариант C)

**Проблема (исходно).** `StoriesOverlay` открывался по `game:start`, но тики на
сервере уже шли. Новичок читал 5 слайдов, пропускал все 5 ходов раунда и погибал
в первом же катаклизме.

**Реализовано:** первый Play запускает **practice-матч с ботом**
(`practice:start`): тики без таймера (тик резолвится по действию игрока),
контекстные подсказки `TutorialHud.vue` вместо слайдов, прогноз 6 с, бот
пропускает ~30% ходов. Матч не пишется в реплеи/статистику, зрители и архитектор
к нему не подключаются. После конца матча ставится `wheee:tutorial_done`
(легаси `wheee:stories_skipped` тоже засчитывается) — дальше Play ведёт в
обычную очередь. `StoriesOverlay` удалён.

## 2. Игрок не видит соперника

Двусторонняя карта — фишка дизайна, но на экране просто пустое поле, и ничто не
говорит, что соперник существует. Кнопка flip-view — маленькая иконка в правом
нижнем углу без подсказки.

Идеи: индикатор состояния соперника (жив/позиция-тень/что делает), заметный
тумблер flip-view с onboarding-тултипом в первом матче, мини-виджет «другая
сторона» (picture-in-picture).

## 3. Поле визуально монотонное, слабая обратная связь по действию

Ровная зелёная плоскость, сетка едва заметна, подсветка подтверждённого действия
(полупрозрачный зелёный) сливается с травой. Высота читается только на сильно
поднятых клетках.

Идеи: контрастнее сетка, цветовая шкала/затенение по высоте, явный чип «моё
действие на этот тик» в HUD + возможность отменить до конца тика.

## 4. Радиальное меню Raise/Lower малоконтрастное

Бледные полупрозрачные кружки с мелкими подписями на зелёном фоне. На мобильном
под пальцем читаемость будет ещё хуже. Поднять контраст, размер, добавить иконки.

## 5. HUD наблюдателя почти невидим и не объясняет механики

«0 PTS» и кнопки Vane/Baro — тусклые квадратики в углу. Нет подсказки, что можно
кликнуть по игроку и предсказывать ходы/победителя. Новый зритель не узнает о
своих механиках. Нужен мини-onboarding вотчера (один тултип-цикл) + контрастнее
кнопки.

## 6. Экран Game Over скупой

Подзаголовок и «Back to lobby» едва читаются. Нет статистики матча (тиков
прожито, причина смерти, изменение статов) — нет «крючка» для Play Again.

## 7. Мелочи в лобби

- Recent-список показывает «Wheat vs Wheat — A won» — имена персонажей вместо
  ников (Bilibin, Lavrushka + флаги уже есть в игре).
- Демо-синематик иногда ловит ракурс «ребром» — карта выглядит серой плитой.
- «No ranked players yet» → заменить на призыв сыграть.

# Решения по п.1 (туториал vs живой матч) — выбран и реализован вариант C

## Вариант A — туториал до очереди (рекомендуется как быстрый фикс)

Клиентское изменение в одном файле (`App.vue`):

- По клику Play: если `wheee:stories_skipped` не установлен — показать
  `StoriesOverlay` поверх лобби-демо (демо как раз показывает механики), очередь
  **не** запускать.
- По done/skip — ставить флаг и запускать очередь с выбранным персонажем.
- Убрать показ stories из обработчика `game:start` (там остаётся только
  `startIntroAnimation()`).

Плюсы: ни одного потерянного тика, нет изменений сервера/протокола, честно для
PvP (оба входят в матч одновременно готовыми). Минусы: туториал вне контекста
реальной доски (смягчается лобби-демо на фоне).

Заодно решить: ставить флаг и по `done`, а не только по `skip` (сейчас туториал
повторяется каждый матч — с вариантом A это станет повтором перед каждой
очередью, что хуже; флаг по done обязателен).

## Вариант B — ready-check перед стартом раунда (серверный, средний)

Добавить `player:ready` в протокол; `Room` не вызывает `beginRound()`, пока оба
игрока не прислали ready (таймаут 20–30 с → автостарт). Инфраструктура пауз в
`Room.ts` уже есть (`pausedTick`/`resumeTimers` для дисконнектов), но чище не
стартовать вовсе, чем ставить на паузу.

Плюсы: туториал можно показывать в контексте реальной доски; заодно получаем
экран «оба готовы» (полезен для будущих фич — пик персонажа, ставки).
Минусы: в PvP соперник ждёт и нужен индикатор «оппонент в туториале»; изменения
протокола + Room + тесты; окно для затягивания матча.

## Вариант C — первый матч = обучение с ботом (большой, правильный onboarding)

Первый матч принудительно против бота в режиме practice: тик не по таймеру, а по
действию игрока; слайды заменяются контекстными степ-подсказками («подними
клетку», «посмотри прогноз», «переверни карту»). После победы/поражения —
обычная очередь.

Плюсы: лучший онбординг, обучение действием, заодно закрывает п.2 (подсказка про
flip-view) и п.5-подобные пробелы. Минусы: заметный объём — режим движка без
таймера, сценарий подсказок, состояние «прошёл обучение» на сервере или в
localStorage.

## Итог

Реализован **вариант C** (см. статус в п.1). Ключевые файлы:
`Room.ts` (practice-режим), `matchmaking`-обход в `index.ts` (`practice:start`),
`TutorialHud.vue`, `App.vue` (роутинг Play → обучение/очередь),
`practice-room.test.ts` (5 тестов).
