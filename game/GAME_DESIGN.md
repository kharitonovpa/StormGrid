# StormGrid — Game Design (MVP)

## 1. Overview

PvP тактическая игра 1v1 на сетке 7×7.
Игроки одновременно (в тиках) перемещаются и изменяют рельеф.
После серии тиков происходит катаклизм (ветер/дождь).
Цель — заставить соперника погибнуть от катаклизма.

---

## 2. Board

* Размер: 7×7
* Координаты: (x, y), где 0 ≤ x,y < 7
* Высота клетки: `-1 | 0 | +1`

    * `-1` — яма
    * `0` — равнина
    * `+1` — холм
* Начальное состояние: все клетки = 0

---

## 3. Players

* Количество: 2 (A и B)
* Начальная позиция: центр (3,3) — оба могут находиться на одной клетке
* Игрок занимает 1 клетку
* Высота игрока = высоте клетки

---

## 4. Time System (Ticks)

* `tickDuration`: 5–7 секунд
* `ticksPerRound`: 5

Цикл:

1. Tick (игроки делают действия)
2. ...
3. Tick (5-й)
4. Weather (катаклизм)

---

## 5. Actions (1 action per tick)

Если игрок не сделал действие — оно сгорает.

### 5.1 Move

* Направления: N, S, E, W
* Смещение на 1 клетку
* Нельзя выйти за карту

### 5.2 Raise

* `height += 1`
* Ограничение: максимум +1

### 5.3 Lower

* `height -= 1`
* Ограничение: минимум -1

---

## 6. Terrain Rules

* Можно изменять любую клетку, **кроме занятой игроком**
* Если на клетке стоит ≥1 игрок — изменение запрещено

---

## 7. Simultaneity

* Все действия применяются **одновременно в конце тика**
* Конфликты:

    * Raise + Lower → нет изменения
    * Move в одну клетку → оба могут стоять вместе

---

## 8. Weather System

Типы:

* Wind
* Rain
* Wind + Rain

Игрок видит **вероятности** до начала раунда.

---

## 9. Wind

### 9.1 Direction

* N→S, S→N, E→W, W→E

### 9.2 Rules (для каждого игрока)

1. Проверка клетки впереди по направлению ветра:

    * если `height(cellAhead) > height(player)` → ветер блокирован

2. Иначе игрок начинает двигаться по ветру:

    * двигается клетка за клеткой

3. При движении:

    * если следующая клетка **выше** → остановка перед ней
    * если **ниже** → игрок падает и останавливается
    * если равна → продолжает

4. Если достигнут край карты:

    * игрок погибает

---

## 10. Rain

### 10.1 Basin Detection

* Низина = область клеток, окружённых клетками строго выше или краем карты
* Край карты считается стеной

### 10.2 Water Fill

* Вода заполняет **минимальный уровень** в каждой низине

### 10.3 Death

* Игрок погибает, если стоит на клетке с этим минимальным уровнем

Примеры:

* Игрок в `-1`, вокруг `0` → смерть
* Игрок в `0`, вокруг `+1` → смерть
* Игрок в `0`, но есть `-1` рядом → выживает

---

## 11. Win / Lose

Игрок проигрывает, если:

* его сдуло за карту (wind)
* он утонул (rain)

---

## 12. Core Gameplay Loop

Повторяется до победы:

1. Показ прогноза погоды
2. 5 тиков:

    * каждый тик: 1 действие
3. Катаклизм
4. Проверка смерти

---

## 13. Design Principles

* Простые правила → сложные ситуации
* Нет мгновенных гарантированных убийств
* Last-tick действия = mindgame, не exploit
* Игрок всегда имеет шанс спастись (move)

---

## 14. MVP Scope

В MVP нет:

* высот > +1 / < -1
* диагонального движения
* особых клеток
* способностей
* мульти-юнитов

---

## 15. Tech Target

* Language: TypeScript
* Engine: custom GameEngine (pure logic)
* Render: Three.js
* UI: Vue 3

---

## 16. Required Engine API (high-level)

```ts
initGame(): GameState

applyTick(actions: {
  A?: Action
  B?: Action
}): GameState

applyWeather(type: WeatherType): GameState

getState(): GameState
```

---

## 17. Data Structures (suggestion)

```ts
type Height = -1 | 0 | 1

type Cell = {
  height: Height
}

type Player = {
  id: 'A' | 'B'
  x: number
  y: number
}

type GameState = {
  board: Cell[][]
  players: Record<'A' | 'B', Player>
  tick: number
  round: number
}
```
